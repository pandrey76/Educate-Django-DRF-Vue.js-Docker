
    1. Не надо в Docker контейнер устанавливать пакет "env", т.е. пакет для создания виртуального окружения, т.к. Docker
сам является конечной платформой и дополнительно устанавливать в него дополнительные виртуальные среды Python не имеет
никакого смысла.
    2. Рекомендуется называть проект свегда просто "project", а потом переименовывать потому, что django обладает
странной структурой он сначало создаёт проект и поддиректорию с одинаковыми именами:
    "project/project" мы переимеуем например в "backend/project".
        3. Нормальный код на JavaScript нельза написать без асинхронцщины.
        4. Тип Decimal нужно использовать в СУБД с осторожностью, т.к. многие СУБД хранят тип Decimal в виде строк.
Соответственно, сортировка по Decimal в таких СУБД будет очень медленной.
